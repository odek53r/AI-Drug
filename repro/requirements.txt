#   ↑ 這個映像【已內含】torch 與 dgl,直接用就好,不必再裝它們。
#
# ── 為什麼不把 torch / dgl 寫進這個檔 ───────────────────────────────────
#   PyPI 上的 `dgl` 是 CPU-only —— pip install dgl 會裝到一個「能 import、
#   但吃不到 GPU」的版本,而且不會有任何警告。ARM64 更是連 CUDA wheel 都沒有。
#   torch 的 CUDA 版也需要特定的 --index-url。
#   把它們寫在這裡只會讓人以為裝好了,實際上跑起來是 CPU(慢 8 倍)或直接炸。
#
#   要自行安裝(x86_64,我們沒測過):
#     torch : https://pytorch.org/get-started/locally/
#     dgl   : https://www.dgl.ai/pages/start.html   ← 選對應的 CUDA 版本
#
# ── 沒有 GPU 也可以跑 ──────────────────────────────────────────────────
#   run_all.py 會自動偵測,沒有 GPU 就退回 CPU(~2 小時 vs GPU ~15 分)。
#   實測 CPU 與 GPU 的 loss/AUC 一致。此時 CPU-only 的 dgl 就夠用。
#
