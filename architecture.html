🏗️ 最佳方法架構:GNN + 雙向傳播 + NMF(堆疊 / late fusion)

三個獨立方法各自跑完 → 把「分數矩陣」標準化後加權相加 → 排名。融合層不訓練,只做算術。
① 共同輸入
藥–病關聯矩陣 Y(894 藥 × 504 病,已知「藥治病」=1)+ 藥相似度 + 疾病相似度
↓ ↓ ↓ 同一份輸入,分給三個分支
② 三個獨立分支(各自訓練/計算,互不干擾)

🔵 分支1:GNN

抓「家族藥」(結構像已知藥)
輸入Y + 異質圖
做法訓練圖神經網路(10 折)→ 藥/病向量 → 內積
→ 分數矩陣 A (894×504)
已訓練好 = resultKPetSup2

🟣 分支2:雙向傳播

跨物種借「相似人類病」的藥
輸入Y + 藥相似度 + 疾病相似度
做法迭代傳播 F = a·(藥相似@F) + a·(F@病相似) + (1−2a)·Y
→ 分數矩陣 B
無訓練,純迭代(20 次)

🟠 分支3:NMF

抓「非結構的重定位藥」
輸入Y
做法矩陣分解 Y ≈ W×H(隱藏因子)→ 重建
→ 分數矩陣 C
無訓練圖,純矩陣分解
↓ A↓ B↓ C
③ 融合層(不訓練,只算術)
每個矩陣按疾病欄 z 標準化(拉到同尺度,否則大的吃掉小的)後加權相加:
綜合 = z(A) + 0.7·z(B) + 1.0·z(C)
為什麼要 z 標準化:A 是 logit、B 是 0~1、C 是重建值,尺度差很多,直接加會被最大的蓋掉。
權重 0.7 / 1.0 怎麼來:掃參數挑驗證最好的(不是先驗知道)。
為什麼不「訓練一個合併器」:只有 121 個標籤,訓練式會過擬合;簡單加權和最穩。

④ 輸出
綜合分數矩陣 → 對每個寵物病把 894 藥排名 → top 候選藥
確認成績:整體 77% / 非人氣 63% / 重定位 22%(vs 原始 GNN 59/41/0)
